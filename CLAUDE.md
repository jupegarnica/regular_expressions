# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
deno task test          # Run all tests (fail-fast)
deno task test-watch    # Run tests in watch mode
deno task test-doc      # Run tests including doc tests
deno task ci            # Lint + format check + tests (full CI)
deno task fmt           # Format code
deno task bench         # Run benchmarks
deno task docs          # Regenerate README from source patterns
deno task build-npm     # Build NPM package via DNT
```

To run a single test file: `deno test src/number.ts test.ts --filter "number"`

## Architecture

This is a Deno regex pattern library. All patterns live in `src/*.ts` and are re-exported from `mod.ts`.

### Pattern file convention

Every pattern in `src/` follows a strict naming convention — each pattern exports four symbols:

```ts
export const patternName: RegExp               // the actual regex (no flags)
export const patternName_description: string   // human-readable description
export const patternName_should_match: string[] // positive test cases
export const patternName_should_not_match: string[] // negative test cases
```

`test.ts` dynamically discovers all exported `RegExp` values and runs three test levels automatically:
1. **"has"** — pattern matches anywhere in a longer string (surrounded by random chars)
2. **"is"** — pattern matches the entire string (`^...$`)
3. **"is not"** — `should_not_match` strings do not match

### README is auto-generated

`README.md` is generated by `_scripts/build_docs.ts` from `deno doc --json` output. Do not edit it manually — run `deno task docs` to regenerate after changing patterns or JSDoc comments.

### Adding a new pattern

1. Add to the appropriate `src/*.ts` file (or create a new one)
2. Follow the four-export convention above
3. Re-export from `mod.ts` if adding a new module
4. Run `deno task test` to validate, then `deno task docs` to update README
